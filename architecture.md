# CRM Application Architecture

---

# Technical Architecture

## Overview

Browser
→ SvelteKit
→ Drizzle ORM
→ PostgreSQL (Supabase)

Authentication:
Browser → SvelteKit → Supabase Auth

File Uploads:
Browser → Supabase Storage
(with signed URLs generated by SvelteKit)

# Authentication

Provider:

Supabase Auth

Flow:

Login
→ Supabase Auth
→ Session Cookie
→ Protected Routes

Protected Pages:

- /dashboard
- /clients
- /finance

---

# Validation

Library:

- Zod

Purpose:

Validate all incoming data before database operations.

---

# Database

Tables:

## Clients

Stores:

- Name
- Email
- Phone
- Notes

Relationship:

One Client → Many Projects

---

## Projects (or Deals)

Stores:

- Title
- Total Amount
- Paid Amount
- Invoice Status
- Payment Status
- Date

Relationship:

One Project → Many Payments

---

## Payments

Stores:

- Amount
- Date
- Note

Purpose:

Track payment history.

---

## Files

Stores:

- Storage Path
- Filename
- File Type
- Upload Date

Only metadata is stored in PostgreSQL.

Actual files live in Supabase Storage.

---

# File Storage

Provider:

Supabase Storage

Bucket:

client-files

Upload Flow:

User
→ Request signed upload URL
→ Upload directly to Supabase Storage
→ Save metadata in PostgreSQL

Files never pass through the application server.

---

# Revenue Calculations

Total Revenue:

SUM(paid_amount)

Outstanding Revenue:

SUM(total_amount - paid_amount)

for projects that are not fully paid.
